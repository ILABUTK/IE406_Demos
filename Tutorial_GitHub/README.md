# Git & GitHub Tutorial Series
### For Undergraduate Students

Welcome! This series will take you from zero Git experience to confidently collaborating on real projects with teammates.

---

## Why Learn Git?

Every professional software team, data science team, and engineering organization uses version control. Git is the industry standard. Learning it now means:

- You never lose work again
- You can collaborate without overwriting each other's files
- Your resume stands out
- You can contribute to open-source projects

---

## How to Use These Tutorials

- **Type every command yourself.** Copy-paste teaches nothing. Muscle memory matters.
- **Make mistakes on purpose.** The best way to learn recovery is to need it.
- **Use a test repository.** Create a throwaway repo for practice so you're not afraid to experiment.

---

## Prerequisites

- A computer running Windows, macOS, or Linux
- A free [GitHub account](https://github.com)
- Basic comfort with your operating system's terminal/command prompt (or use GitHub Desktop — see Tutorial 08)

---

## Quick Reference Card

```
git init              Start a new repo
git clone <url>       Download an existing repo
git status            See what changed
git add <file>        Stage a file for commit
git commit -m "msg"   Save a snapshot
git push              Upload commits to GitHub
git pull              Download latest changes
git branch <name>     Create a new branch
git checkout <name>   Switch branches
git merge <name>      Merge a branch into current
git log               View commit history
git diff              See line-by-line changes
```
